# 社区

欢迎来到 A2UI 社区！我们正在共同构建代理驱动界面的未来。

## 参与其中

A2UI 是一个基于 Apache 2.0 许可的开源项目。我们欢迎开发人员、研究人员以及任何有兴趣推进代理用户界面的人做出贡献。

## 社区展示

    我们正在考虑如何最好地展示社区项目、示例、主题、渲染器、自定义组件等。在 Github 讨论区链接一个 4 分钟（或更短）的演示视频和代码示例是展示你作品的好方法。

## 项目合作伙伴

A2UI 由多个组织合作开发：

### Google Opal

[Opal](http://opal.google) 让用户可以使用自然语言构建、编辑和分享 AI 迷你应用程序。Opal 团队一直是 A2UI 开发的核心贡献者。

### Flutter

[Flutter 的 GenUI SDK](https://docs.flutter.dev/ai/genui) 使用 A2UI 作为 UI 声明格式，用于在 Flutter 应用程序中生成动态的、个性化的 UI。

### Gemini Enterprise

A2UI 正在被集成到 [Gemini Enterprise](https://cloud.google.com/gemini-enterprise?e=48754805) 中，以使自定义代理能够在企业应用程序中渲染丰富的、交互式的 UI。

### AG UI / CopilotKit

[AG UI](https://ag-ui.com/) 和 [CopilotKit](https://www.copilotkit.ai/) 提供了与 A2UI 的零日兼容性，使开发人员能够构建丰富的、状态同步的应用程序，从而渲染来自代理的动态 UI。

### A2A

Google 的 [A2A 团队](https://a2a-protocol.org/) 一直是 A2UI 开发的核心贡献者，并得到了 A2A 技术指导委员会 (TSC) 的支持。

### ... 以及更多

还有许多组织和个人为 A2UI 的开发做出了贡献。

如果你对 A2UI 做出了重大贡献，请将你的名字提交到此列表。

## 行为准则

我们致力于为每个人提供一个受欢迎和包容的环境。所有参与者都应：

- 尊重和体谅他人
- 欢迎新人并帮助他们入门
- 专注于对社区最有益的事情
- 对他人表现出同理心

我们绝不容忍不可接受的行为。请向项目维护者报告疑虑。

## 致谢

我们感谢所有的贡献！贡献者将在以下位置获得认可：
