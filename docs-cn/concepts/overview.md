# 核心概念

本节解释了 A2UI 的基本架构。理解这些概念将帮助你构建高效的代理驱动界面。

## 宏观图景

A2UI 围绕三个核心理念构建：

1. **流式消息**：UI 更新以一系列 JSON 消息的形式从代理流向客户端
2. **声明式组件**：UI 被描述为数据，而不是被编写为代码
3. **数据绑定**：UI 结构与应用程序状态分离，从而实现响应式更新

## 关键主题

### [数据流](data-flow.md)
消息如何从代理传输到渲染的 UI。包含餐厅预订流程的完整生命周期示例、传输选项（SSE、WebSockets、A2A）、渐进式渲染和错误处理。

### [组件结构](components.md)
A2UI 用于表示组件层级结构的 **邻接表模型**。了解为什么扁平列表优于嵌套树，如何使用静态与动态子级，以及增量更新的最佳实践。

### [数据绑定](data-binding.md)
组件如何使用 JSON Pointer 路径连接到应用程序状态。涵盖响应式组件、动态列表、输入绑定以及使 A2UI 如此强大的结构与状态分离。

## 消息类型

A2UI 使用四种消息类型：

- **`surfaceUpdate`**：定义或更新 UI 组件
- **`dataModelUpdate`**：更新应用程序状态
- **`beginRendering`**：通知客户端进行渲染
- **`deleteSurface`**：移除 UI 界面

有关完整的技术细节，请参阅 [消息参考](../reference/messages.md)。
